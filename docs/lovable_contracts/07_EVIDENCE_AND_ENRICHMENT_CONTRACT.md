# 07 — Evidências e enriquecimento

## Documento de evidências

Explica por que um valor foi aceito, corrigido ou rejeitado. Exemplos: evidência física, consenso, matemática, coverage, fragmentos.

## Documento de enriquecimento

Sugere melhorias futuras para o sistema:

- unidades candidatas;
- aliases de banco;
- padrões de código;
- headers recorrentes;
- templates de tabela;
- ruídos rejeitados.

Nunca autoaplica em base_config. Admin/usuário precisa aceitar.
