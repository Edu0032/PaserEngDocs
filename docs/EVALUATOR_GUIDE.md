# Guia rápido para avaliação

## O que observar primeiro

1. `README.md` para visão geral.
2. `parser_browser/app/browser/` para entrada Pyodide/browser.
3. `parser_browser/app/parser/integrity_orchestrator.py` para fluxo principal.
4. `api_docling/app/` para API FastAPI com Docling.
5. `schemas/` para contratos JSON.
6. `tests/` para validação de regressão e estabilidade.

## Competências demonstradas

- Python aplicado a parsing de documentos.
- Integração entre frontend, Pyodide e API.
- Validação e organização de dados em JSON.
- Contratos de entrada e saída.
- Tratamento de PDFs, tabelas e evidências.
- Testes automatizados e documentação técnica.
- Separação entre parser, API e contratos.

## Relação com vaga de tecnologia/dados

O projeto demonstra automação em Python, validação de dados extraídos de PDFs, integração com API, organização de dados estruturados, documentação de processo e atenção a detalhes em documentos complexos.
